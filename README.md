# AB-RoPE for Sequential Recommendation


This repository implements **AB-RoPE** (Adaptive Behavior-aware Rotary Positional Embedding), an enhancement for the **HSTU** (Hierarchical Space-Time Unit) architecture in sequential recommendation. It specifically addresses the integration of multi-behavioral data (PV, Favorite, Cart, Buy) and temporal dynamics using modified Rotary Positional Embeddings.

## Reproducibility: Ablation Studies

This project includes 6 experimental configurations to validate the components of the full model. You can run specific experiments by passing the `--exp_id` flag.

| **Exp ID** | **Configuration**   | **Description**                                     |
| ------------------ | --------------------------- | ----------------------------------------------------------- | 
| 1                | `HSTU + Std RoPE`     | Standard HSTU with basic Rotary Positional Embeddings.    |
| 2                | `HSTU + Time Bias`    | HSTU with time-aware attention bias (no RoPE).            |
| 3                | **Baseline**             | HSTU + Standard RoPE + Time-aware attention bias.         |
| 4                | `Baseline + Dyn-Freq` | Baseline + Dynamic-frequency-only-RoPE.                   |
| 5                | `Baseline + Behavior` | Baseline + Behavior-only modulation.                      |
| 6                | **Full Model**           | **Baseline + AB-RoPE (Full behavior/time integration).** |

## Running the Experiments

The `Main.py` script serves as the unified entry point for both training and evaluation.

#### **Command Line Arguments**

| **Argument**     | **Type** | **Default** | **Description**                                                 |
| ------------------------ | ---------------- | ------------------- | ----------------------------------------------------------------------- |
| `--exp_id`         | `int`      | `6`           | Selects the ablation study configuration (1-6).                       |
| `--inference_only` | `flag`     | `False`       | Skips training and runs evaluation on the test set.                   |
| `--ckpt_path`      | `str`      | `""`          | Path to a`.pth`file. Defaults to`./result/exp_{id}/best.pth`. |

#### **1. Training**

To train the full model (Exp 6):


```bash
python Main.py --exp_id 6
```

Logs and checkpoints are saved under `./result/exp_6/`.

#### **2. Evaluation (Reproducing Report Results)**

To reproduce the performance metrics (HR@10 and NDCG@10) from the report, run:


```bash
# Evaluate the Full Model (Exp 6)
python Main.py --exp_id 6 --inference_only

# Evaluate the Baseline (Exp 3)
python Main.py --exp_id 3 --inference_only

# To evaluate a specific checkpoint file located elsewhere, provide the --ckpt_path
python Main.py --exp_id 6 --inference_only --ckpt_path ./your_custom_path/best.pth
```

The script will automatically load the corresponding weights from the `result/exp_X/` directory.
